18. Feature Inventory Explorer (index.html)

Step 22: Open the feature inventory overview

Start on the inventory landing view to review summary chips, filters, and role-grouped feature cards.

How to use: Open features/index.html, review the summary metrics, then scroll into role sections to inspect feature cards.

Accessibility: The role results container uses a live region (aria-live="polite") so assistive tech receives updates when filters change.

Troubleshooting: If metrics stay at 0, refresh the page and confirm JavaScript is enabled in the browser.

Step 23: Filter features with the search field

Use the keyword search field to narrow features by name, route, description, permission text, or role labels.

How to use: Click Search Features, type a keyword such as billing, and review filtered cards and updated counts.

Accessibility: Keyboard users can press Tab to focus the field and type directly; screen readers announce the labeled input.

Troubleshooting: If no cards appear, clear the query or check whether role/status filters are also restricting results.

Step 24: Use role-based filtering

Role filtering limits visible cards to features available for a selected user role.

How to use: Open Role Filter, choose a role like Company Admin, then review the updated cards and counts.

Accessibility: Native select controls support keyboard arrows, Home/End, and Alt+Down (browser dependent).

Troubleshooting: If the selected role is not visible, reset filters to All Roles and reload once to repopulate options.

Step 25: Filter by implementation status

Status filtering isolates implemented, partial, or planned entries for audit or release checks.

How to use: Select Partial or another status from Status Filter and compare the metric cards.

Accessibility: Status labels also appear as colored chips on each card, improving at-a-glance recognition.

Troubleshooting: If expected items do not appear, verify their current status values in features/index.html.

Step 26: Include or exclude micro-features

The micro-feature toggle helps separate major workflows from smaller UI utilities.

How to use: Check Include micro-features to include helper/UI-level items in the role sections.

Accessibility: The checkbox is label-bound and can be toggled using Space when focused.

Troubleshooting: If counts do not change, clear search/status filters first to make the effect visible.

Step 27: Validate empty-state behavior

When filters eliminate all results, the page provides a clear empty-state message.

How to use: Enter an uncommon term (for example zzzz-no-match) to verify graceful no-result behavior.

Accessibility: The empty-state text is plain, high-contrast content that remains readable without hover interactions.

Troubleshooting: Remove the search term and reset dropdown filters to restore the full feature list.

Step 28: Expand and collapse role sections

Role sections are rendered as expandable groups so large inventories remain scan-friendly.

How to use: Click a role header to expand its card grid, then click again to collapse that section.

Accessibility: Each role group uses semantic <details>/<summary> elements with keyboard support (Enter/Space).

Troubleshooting: If a section appears empty, confirm filters still include at least one matching feature for that role.

Step 29: Navigate controls by keyboard

All primary controls can be reached and operated without a mouse.

How to use: Press Tab and Shift+Tab to move focus, then use Enter, Space, and arrow keys to interact with controls.

Accessibility: Form labels are explicitly associated with each field and controls preserve native keyboard behavior.

Troubleshooting: If focus indicators are hard to see, test in a browser without custom high-contrast overrides first.

Step 30: Verify responsive mobile layout

The inventory is responsive and reorganizes controls, cards, and metrics on narrow screens.

How to use: Resize the browser window (or use device emulation) and validate stacked controls and readable cards.

Accessibility: Responsive breakpoints keep content linear and scannable for zoomed or mobile contexts.

Troubleshooting: If layout remains desktop-width, clear browser zoom overrides and ensure responsive viewport meta tags are active.

Step 31: Use quick guide links from the hero chips

The hero area includes direct links to both the HTML and Markdown guides.

How to use: Select How-To Guide or Full User Guide (Markdown) chips to open documentation directly.

Accessibility: Link chips use text labels (not icon-only controls), improving discoverability for screen readers and keyboard users.

Troubleshooting: If links fail, verify all guide files remain under the features folder with unchanged filenames.

Note: This guide references screenshots captured from the current environment and validates linked image paths at generation time.
