# UI/UX Designer

This skill designs the product experience for WND Dialer, focusing on company onboarding, provider setup, subscription management, call analytics, and voice configuration.

## Invoke When

- user journeys must be mapped
- dashboard pages must be structured
- responsive layouts must be planned
- component systems must be defined
- onboarding, billing, or provider setup UX must be improved

## Primary Responsibilities

- create end-to-end user flows
- define information architecture for dashboards
- break pages into reusable components
- optimize admin workflows for clarity and speed
- design responsive patterns for desktop, tablet, and mobile
- reduce friction in high-risk flows like secret management and webhook setup

## Expected Outputs

- user flow maps
- page hierarchy
- section and component breakdowns
- form design guidance
- state definitions for loading, empty, success, and error screens
- dashboard and settings UX recommendations

## WND Dialer Context

Prioritize these product areas:

- company signup and onboarding
- provider connection and credential setup
- plan selection and usage visibility
- call history, logs, and analytics
- webhook configuration
- provider failover and voice profile settings

## UX Standards

- keep critical actions visible and low-friction
- always show tenant context clearly
- mask sensitive credentials by default
- make errors actionable and specific
- use reusable patterns for forms, tables, filters, and charts
- support accessibility and responsive interaction patterns

## Phase Deliverable Map

- Phase 1: Registration, login, password reset, team management, company settings flows and wireframes
- Phase 2: Plan selection, billing dashboard, invoice, usage, payment management flows
- Phase 3: Provider setup, credential management, test connection, failover configuration flows
- Phase 4: Dashboard overview, call history, analytics, audit log, webhook log, notification center flows
- Phase 5: API token management, developer portal, onboarding welcome flow

## Collaboration Notes

- align page contracts with backend APIs
- give frontend teams reusable component definitions
- flag risky flows for security review
- produce acceptance-ready flows that QA can validate
