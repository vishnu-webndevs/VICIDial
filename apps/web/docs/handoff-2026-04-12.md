## What Was Fixed
1. API unauthenticated exception flow (`Route [login] not defined`).
   - File: `apps/api/bootstrap/app.php`
   - Change:
     - Added API-safe `AuthenticationException` JSON response handling in `withExceptions(...)`.
     - Added guest redirect fallback wiring in middleware configuration.
   - Outcome: onboarding/auth flows no longer fail due to login route fallback in API requests.

2. Intermittent login bootstrap failure in E2E (timeout / non-JSON response handling).
   - File: `apps/web/e2e/full-app.e2e.spec.ts`
   - Change:
     - Hardened login helper with retries, backoff, longer request timeout.
     - Added `Accept: application/json` on auth calls.
     - Added guarded JSON parsing with clear failure message for non-JSON payloads.
   - Outcome: auth bootstrap and analytics+billing scenarios are stable.

3. Visual regression instability (small diff noise + occasional width drift).
   - File: `apps/web/e2e/visual-regression.spec.ts`
   - Change:
     - Forced vertical scrollbar for stable viewport width.
     - Added `maxDiffPixelRatio: 0.015` tolerance for renderer-level noise.
     - Refreshed snapshots for current environment.
   - Outcome: visual suite passes consistently in reruns.
